Cards are containers that display content about a single subject. They are also useful for grouping a list of related information.

In its definition cards are containers that need to include other components like list/items, images, buttons avatars etc.

Default

Flags

A card can show a flag in the very top. This is mostly used when a bundle of items in the card can express the same “flag”.

Colors

Cards can have a colored background to stand out even more.

Background image

The recommended approach for setting a background image on the card is with the --kirby-card-background-image CSS Custom Property.

Note: this example uses include-media for media queries. This is re-exported from the @kirbydesign/designsystem/scss/utils Sass module.

State & Disclosure

The card expresses its state with its container color shade & tint, when hovered or pressed. Registering any click event (click)="function()" on the card will activate its state behavior.

Whenever a card is clickable it is a best practice to also add a disclosure indicator.

A card with a disclosure indicator is composed of a card-header with a slotted kirby-item, and often uses --kirby-card-padding-top CSS Custom Property to set the padding top in card content area to 0px.

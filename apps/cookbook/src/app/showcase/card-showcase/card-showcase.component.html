Cards are containers that display content about a single subject. They are also useful for grouping a list of related information.

In its definition cards are containers that need to include other components like list/items, images, buttons, avatars etc.

The card supports two major use-cases:

  • The default mode is useful when a card is used purely for layout and grouping purposes.
  • Whenever a card is used for navigation, for example to a new page, drawer or modal the disclosure-variant should be used.

State & Disclosure

Whenever a card can be interacted with it is a best practice to also add a disclosure indicator, to show that it navigates to a new page, opens a modal or similar.

Registering any click event (click)="function()" on the card will activate its state behavior when hovered or pressed.

A card with a disclosure indicator is composed of a card-header with a slotted kirby-item , using the --kirby-card-padding-top CSS Custom Property to set the padding top in card content area to 0px . The card header input property hasPadding is set to false to adjust to the use of item.

Flagged header

A card can show a flag in the header. This is mostly used when a bundle of items in the card can express the same “flag”.

Colors

Cards can have a colored background to stand out even more.

Background image

The recommended approach for setting a background image on the card is with the --kirby-card-background-image CSS Custom Property.

Alternatively, it can be set programatically via the backgroundImageUrl input property.

In this example, themeColor is explicitly set to dark to tell the card that the image content is dark. This ensures that the hover and pressed state make the card lighter instead of darker.

Note: this example uses include-media for media queries. This is re-exported from the @kirbydesign/designsystem/scss/utils Sass module.

Flat

By default the card has a slightly elevated look with a shadow around its edges. This can be removed by setting the flat input, but be aware that this option should not be used with a card that can be interacted with (clickable, disclosure) as it lacks proper affordance.

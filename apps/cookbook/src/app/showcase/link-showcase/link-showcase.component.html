Links are used for navigation between pages. They can be standalone links used on their own or inline links placed within a sentence.

A link should be used for simple navigation. That is, situations where the user will be sent to a new resource or destination in the app via a href. Use buttons instead of links if the user is not redirected to a new destination but instead triggers an action, makes a choice, or in any way manipulate data.

Rather than providing a component for links Kirby simply provides custom styles for the <a> tag that gives a consistent look and feel of links. Links have an underline and inherit the text color of the surrounding elements. The color of visited links does not change.

Sizes

By default, links inherit the font-size of the context they are used in. For standalone links, any font-class from Kirby Fonts can be applied. For accessibility reasons it is not recommended to use links when the clickable text is smaller than 12px (extra small).

New tab

The default behavior is to open the link in the current tab. If an anchor-tag has target="_blank" it will get an icon to signal to the user that a new tab will open.

It is recommended to only use this option for standalone links that are not used within a paragraph or in any other way embedded in text. Consider writing out the source for inline links so it is clear that clicking the link will take the user to a new context, e.g. www.github.com/kirbydesign/designsystem. Then the user can choose how they want to open the link.

Note: In a non-web context, i.e. an app, any link will most likely launch the device's browser and effectively open a new tab outside the app context.

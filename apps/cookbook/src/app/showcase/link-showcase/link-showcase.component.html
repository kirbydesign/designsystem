Links are used for navigation between pages. They can be standalone links used on their own or inline links placed within a sentence.

A link should be used for simple navigation. If a user triggers an action, makes a choice, or in any way manipulate data, use a button instead.

Rather than providing a component for links, Kirby simply provides custom styles for the <a> tag, that gives a consistent look and feel of links. Links have an underline, and inherit the text color of the surrounding elements. The color of visited links does not change.

Sizes

Any font-class from Kirby Fonts can be used with links. For accessability reasons, it is not recommended to use links when the clickable text is smaller than 12px (extra small).

New tab

The default behavior is to open the link in the current tab. If an anchor-tag has target="_blank", it will get an icon to signal to the user that a new tab will open.

It is recommended to only use this option for standalone links, that are not used within a paragraph or in any other way embedded in text. For inline links, consider writing out the source, so it is clear that clicking the link will take the user to a new context, e.g. www.github.com/kirbydesign/designsystem. Then the user can choose how they want to open the link.

Kirby comes pre-packaged with a set of icons hand-crafted to be round and friendly. They are used together with other components to explain actions and functions of the UI.

Setup

To setup the Kirby icons ensure that the .svg files used by Kirby are copied to your output folder as explained in our readme file

If the icons exists at another location, it is possible to configure the path that the icons are loaded from by providing the BUILT_IN_ICONS_URL injection token.

The icon component only request icons visible to the the user (icons above the fold). The component will dynamically load an SVG for each icon, so your app is only requesting the icons that you need.

Custom Icons

For adding custom icons, use IconRegistryService in your NgModule . We recommend registering icons that are reused across an application in a top-level module. If icons are registered in modules that are not shared across the application consider namespacing them, e.g. my-feature-custom-icon-name to avoid collisions.

Once registered custom icons can be used via the name property similar to default icons.

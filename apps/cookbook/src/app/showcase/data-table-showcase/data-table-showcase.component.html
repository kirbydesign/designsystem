Data Tables allow the consumer to show large amounts of complex data in tabular form, with rows and columns. Columns can be named through the use of headers.

Please note: Kirby Data Table is not a finished component, and it is very modular. This means that it lacks a number of features and all uses should be thoroughly aligned with a designer before use (use the corresponding design guidelines).

Prominent features that are missing include:
  • EventEmitters on headers
  • Selectable headers
  • Asc/desc sorting arrows in headers
  • EventEmitters on rows
  • Responsivity (i.e., no scroll/collapsible columns/etc.)

Radio button is a control that allows the user to select one option from a set while seeing all available options.

Radios should be used inside of a kirby-radio-group. Pressing on a radio will check it. They can also be checked programmatically by setting the value property of the parent kirby-radio-group to the value of the radio.

When radios are inside of a radio group, only one radio in the group will be checked at any time. Pressing a radio will check it and uncheck the previously selected radio, if there is one. If a radio is not in a group with another radio, then both radios will have the ability to be checked at the same time.

Simple data binding

<kirby-radio-group> supports data binding using simple @Input property and @Output event binding. You can supply an array of any data type through the items property - just remember to also set the itemTextProperty to the member of your data that you want to show as the label (defaults to 'text').

To pre-select an option you can set the selected item in the value property of <kirby-radio-group>.

To disable an option you can set the itemDisabledProperty to the member of a boolean property in your data (defaults to 'disabled').

To be notified when the selected value changes you can bind the change event of <kirby-radio-group>.

Custom radio template

If you need to customize the radio buttons you can supply a custom item template through the *kirbyListItemTemplate directive. Start by wrapping each <kirby-radio> in a <kirby-label> to ensure proper accessibility and alignment of the radio button and its label - this also ensure the entire label is clickable.

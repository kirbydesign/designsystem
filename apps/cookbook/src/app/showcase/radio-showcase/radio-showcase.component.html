Radio button is a control that allows the user to select one option from a set while seeing all available options.

Radios should be used inside of a kirby-radio-group. Pressing on a radio will check it. They can also be checked programmatically by setting the value property of the parent kirby-radio-group to the value of the radio.

When radios are inside of a radio group, only one radio in the group will be checked at any time. Pressing a radio will check it and uncheck the previously selected radio, if there is one. If a radio is not in a group with another radio, then both radios will have the ability to be checked at the same time.

Simple data binding

<kirby-radio-group> supports data binding using simple @Input property and @Output event binding.

You can supply an array of any data type through the items property - just remember to also set the itemTextProperty to the member of your data that you want to show as the label (defaults to 'text').

To pre-select an option you can set the selected item in the value property of <kirby-radio-group>.
This can also be achieved by setting the index of the selected item in the selectedIndex property.

To disable an option you can set the itemDisabledProperty of <kirby-radio-group> to the member of a boolean property in your data (defaults to 'disabled').

To be notified when the selected value changes you can bind the valueChange event of <kirby-radio-group>.

Custom radio template

If you need to customize the radio buttons you can either:

  1. Slot any number of <kirby-radio> between the opening and closing tags of <kirby-radio-group> in your template.
    Please note: In this case you should omit setting the items property of <kirby-radio-group>.
     
  2. Supply a custom item template through the *kirbyListItemTemplate directive. Just remember to include a <kirby-radio> inside your template with the value property set to your data item and the text property set to the textual representation of your data item.

In Form Field

Radios can placed within a <kirby-form-field> and assigned a label and an optional message.
Clicking the label will focus the selected radio, if any, otherwise the first radio.

Validation
If validation for the field fails you can set the hasError property of <kirby-radio-group> (and optionally an error message through the message property of <kirby-form-field>).

ngForms

kirby-radio-group implements the control value accessor interface, which means it is ngForms-enabled. This allows you to use it directly in template driven and reactive forms.

Reactive Forms

To use radios in a reactive form, bind your form control to <kirby-radio-group> with simple data binding:

<kirby-radio-group [formControl]="yourFormControl"> // or simply: <kirby-radio-group formControlName="yourFormControlName">

Template Driven Forms

To use radios in a template driven form, simply two-way bind your selected value to <kirby-radio-group> with ngModel:

<kirby-radio-group [(ngModel)]="selected">

The kirby list will emit a loadMore event, when the user has scrolled 80% of the list. It is the responsibility of the consumer to add more items to the bound items.

The event emitted is of type KirbyLoadMoreEvent which exposes a complete() callback function.

The complete() callback function should be called when the loading of the additional items are done.

If complete(disableLoadMore?: boolean) is called with the disableLoadMore argument set to true, no further events will be emitted. If noMoreItemsText is configured it will be shown at the end of the list at this point.

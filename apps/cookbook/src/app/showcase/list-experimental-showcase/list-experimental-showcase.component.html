Lists are made up of multiple rows of items which can contain items, text, buttons, icons, and much more. Lists generally contain items with similar data content, such as images and text.

The list itself is essentially a container with styling for rounded corners and shadow around the list content. Anything can be placed into the list, but it comes with a bit of extra functionality when used together with kirby-section-header and kirby-item .

When using kirby-item , either on its own or wrapped in custom markup, the list expects its children to represent single item (e.g. not another container or wrapper element). This way, the first and last item will correctly have top and bottom spacing applied, to provide the correct appearance of the list.

List with items

The Section Header can be used on a list to create a set of grouped content, making use of the lists slots .

The list has two slots , the default and the named outside slot. The default slot is reserved for the list items, and receives the list-specific styling. The outside slot is placed outside the styled part of the list, and is meant to be used with the kirby-section-header or similar.

Sliding items in list

Lists work out of the box with the interactive Item Sliding , allowing to swipe an item left or right to reveal options. Try dragging right on one of the items, to see swipe actions revealed to the left of the item content. See the page linked above for more details on kirby-item-sliding .

Sections

It is important to note that the list does not have any functionality for multiple sections nested within the list itself. This would add extra (unnecessary) complexity to using the list.

Lists are made up of multiple rows of items which can contain text, buttons, toggles, icons, thumbnails, and much more. Lists generally contain items with similar data content, such as images and text. The Section Header can be used on a list to create a set of grouped content.

The list itself is essentially a container with the correct styling for rounded corners and shadow around the list content. Anything can be placed into the list, but it comes with a bit of extra functionality when used together with kirby-section-header and kirby-item.

When using the kirby-item, either on its own or wrapped in custom markup, the list expects each of its children to represent a single item (e.g. not another container or wrapper element). This way, the first and last kirby-item will correctly have --item-padding-top and --item-padding-bottom applied, to provide the correct look and feel of the list.

List with items

The list has two slots, the default slot and the named slot list-header. The default slot is reserved for the list items, and receives the aforementioned styling. The list-header slot is placed outside the styled part of the list, and is meant to be used with the kirby-section-header or similar.

Sliding items in list

Lists work out of the box with the interactive Item Sliding, allowing to swipe an item left or right to reveal options. Try dragging right on one of the items, to see swipe actions revealed to the left of the item content. See the page linked above for more details on kirby-item-sliding.

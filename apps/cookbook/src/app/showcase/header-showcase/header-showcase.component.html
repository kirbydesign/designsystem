Header is a component that manages the layout of the header section of kirby-page . It is responsive and will change the layout according to the device size. It allows for a wide range of use cases including support for avatar, flags etc.

Default

Flag

A flag can be used to indicate a status. To use a flag you simply add a <kirby-flag> in the header which will position the flag correctly and manage spacing.

Key value

Scaling of title

Titles are default rendered as h1 .

It's possible to enable auto-scaling of the header title. This will scale both font-size & line-height between h1 -> h3 depending on amount of text and maximum number of lines configured. If the text cannot fit within the configured number of lines it will be truncated using an ellipsis ( … ).

This can be configured by setting the titleMaxLines="NUMBER OF MAXIMUM LINES" attribute of kirby-header .

Actions

Actions can be included in the header area and will by default always be shown in the toolbar on small screens. The actions place themselves in the toolbar to the right. It is designed to optimize the screen real estate for the content area on mobile phones.

On larger screen sizes the actions starts out in the header area, before moving to the toolbar when the page is scrolled past the actions.

Actions can be suppplied through:

  • Action Group component - common use case
  • Custom Actions - complex scenarios

Please note: In both cases the element containing the actions (e.g. kirby-action-group or an ng-container ) needs to be decorated with the *kirbyHeaderActions directive for the page and header components to render the actions in both locations.

Action Group

In most use cases actions should be suppplied using the kirby-action-group component as shown below. This component has sensible defaults for showing the primary action(s) as button(s) and collapsing the rest of the actions into a "more" menu.

Actions are supplied to the Action Group via slotted buttons. Only two actions are shown in the toolbar, and if more that two actions are supplied, one action and a "more" menu button will be shown, while the remaining actions are collapsed into the "more" menu.

All button interactions are expected to happen via a click-listener of type (click)="someFunction()" so the Action Group can make sure to fire the corresponding function for collapsed actions inside the "more" menu.

Header Action Group

Please note: A follow-up release will soon allow the action group to be placed in the header area on small screens and expand beyond two options when space permits for all screen sizes. This additional feature can simply be enabled on examples such as the one above once it lands.

Custom Actions

When more control is needed over how actions are shown the *kirbyHeaderActions directive can be applied to the custom actions element itself - e.g. a kirby-menu when only secondary actions are needed.

When several action elements needs to be combined they can be grouped in an ng-container which is then decorated with the *kirbyHeaderActions directive.

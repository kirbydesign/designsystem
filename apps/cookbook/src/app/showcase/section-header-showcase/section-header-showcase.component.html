The kirby-section-header can be used as a header for a standalone grouping of elements.

The Section Header consists of three possible columns: the default , which spans the entire Section Header, and optionally the 'start' and 'end' columns (aka 'slots' ).

The default column is the content that isn't slotted into either 'start' or 'end' .
If there is more than one line of content it can be contained within a kirby-label that by default stacks content vertically (multi lines).

To place content in the start column specify the slot="start" attribute on your content. If no content is placed into the 'start' slot, the first column will not be shown.

The end column is usually used for values such as number, currency, date etc. To place content in the end column specify the slot="end" attribute on your content. If multiple lines are needed the values can be wrapped in a kirby-label .

Heading, label and detail

The heading , label and detail attributes can be used on the elements slotted inside the Section Header, to apply the default styling to any heading or paragraph.

This means that you are free to use the correct heading element for your context, e.g. h1 if it is the only heading on your page, or as it is the case in these examples a h3, as our section headings are nested within a h1 and h2.

Custom section header

For use cases where styling via heading , label and detail attributes are not sufficient, it is possible to simply supply any markup or component within the section header tags, to render it in the default slot.

The section header is ideal as a first element within a kirby-item-group , as a header for that specific group of items, but can also be used for other groups of content such as a kirby-card .

Page is a layout container for a Kirby page. There should only be one <kirby-page> layout container per page.
The page component gives you the following:

  • Scrollable page title and actions:
    • Sticks to the toolbar by default, when scrolling the page below the title
    • Also supports fixed page title and actions
  • Back navigation
  • Simple configuration for basic page title and actions
  • Support for advanced page title and actions template
  • Support for fixed page content (e.g. a Floating Action Button)
  • Spacing of content

Life-cycle hooks

Be aware that using page with kirby-router-outlet, causes the views to be stacked instead of replaced. This means that you can't trust onInit and onDestroy. Instead you can trust the kirby-page events (enter) and (leave) instead.

Title Alignment & Alternative Toolbar Title

Titles are default left aligned but can be centered (titleAlignment="center") or right aligned (titleAlignment="right").

It also possible to supply an alternative toolbar title (toolbarTitle="A Different Title") - that is shown instead of the page title - when the page title is scrolled above the toolbar.

This can also be achieved through the *kirbyPageToolbarTitle directive.

Advanced Page

For more advanced scenarios, you can supply a custom page title element, action buttons and fixed content with the *kirbyPage-directives

*Please note: When supplying a custom page title element you must supply both a page title and a toolbar title. You should also ensure the page title is wrapped in a h1 tag. See example below on how to reuse a template for both titles.

Fixed Title & Actions

If the toolbarTitle property is specified instead of the title property, the page title and actions will be fixed and shown only in the toolbar and not on the page itself.

Inside the parent (caller) component:

To show the modal, create a ModalConfig and pass it to the ModalController.showModal: constructor(private modalController: ModalController) {{ '{}' }} showModal() {{ '{' }} const config: ModalConfig = {{ '{' }} title: 'Your Modal Title', flavor: 'modal', component: YourEmbeddedModalComponent, componentProps: {{ '{' }} prop1: 'value1', prop2: 'value2' {{ '}' }} {{ '}' }}; this.modalController.showModal(config); {{ '}' }}

The modal can have one of multiple flavors, used to determine the opening animation, the position of their titles and their close buttons. With the modal flavor, the dialog opens with a fade-in animation, its title is aligned to the left and its close button is placed in the top right corner. When a drawer flavor is used, the dialog opens with a slide-up animation, its title is centered and its close button is placed in the top left corner. Additionally, with the drawer flavor, users can specify a drawerSupplementaryAction, which is an additional button rendered in the top right corner (see the example below). The drawerSupplementaryAction option takes an iconName: string which corresponds to an icon of the kirby font, and action: Function, which is invoked when the supplementary button is selected. showModal() {{ '{' }} const config: ModalConfig = {{ '{' }} title: 'Your Drawer Title', flavor: 'drawer', drawerSupplementaryAction: {{ '{' }} iconName: 'qr', action: this.onSupplementaryActionSelect, {{ '}' }}, component: YourEmbeddedDrawerComponent {{ '}' }}; this.modalController.showModal(config); {{ '}' }} private onSupplementaryActionSelect() {{ '{' }} console.log('Supplementary action selected'); {{ '}' }}

(Optional)
If you need to obtain data back from the modal, you can pass a callback function: this.modalController.showModal(config, myCallback); myCallback(dataReturnedByModal: any) {{ '{' }} ... {{ '}' }}


Inside the embedded component:

To create a embedded component, ensure:

  1. the embedded component must be declared (module file, or inside your @Component decorator)
  2. the embedded component must be listed in your entryComponents (module file)

(Optional)

To access data passed into the embedded component, inject the COMPONENT_PROPS inside the embedded components constructor:

constructor( @Inject(COMPONENT_PROPS) private componentProps, private modalController: ModalController, ) {{ '{' }} this.props = componentProps; {{ '}' }}

In order to hide a modal from within your embedded component (and destroy its component), you can use: constructor(private modalController: ModalController) {{ '{}' }} this.modalController.hideTopmost();

(Optional)
You can additionally pass data, which will be available in your callback: let returnData = ...; this.modalController.hideTopmost(returnData); It is possible to nest more modals from within your embedded components,
by using the same services and principles as above.

Opening multiple modals from the same view

In response to issue #385. It might seem compelling to use callbacks from a modal to instruct the "parent" component (the one launching the modal) to launch a secondary modal - this way you would be able to control multiple modals from the same parent-component, which would be nice, but sadly - its not possible.

Conclusion must be: If you want to open a second modal - let the first modal do it - just like the example above shows.
